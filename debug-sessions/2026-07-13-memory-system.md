**Project**: ST-SevenDaysCal
**Repo**: git@github.com:atonal519/ST-SevenDaysCal.git (master)

## Session goals

用户反馈几个问题连续处理：
1. 手动刷新事件线时输出和之前几乎一样（触发路径漏清缓存）
2. 平行事件太多"废话事件线"，缺乏挖掘伏笔的能力
3. 校园/日常卡不需要宏大叙事，需要按叙事尺度分档
4. 长期对话（100+ 楼）会脱离早期剧情，需要摘要式记忆系统

## Changes

### 1. `triggerGenerateLines` 补清缓存
之前手动刷新时 `previousRaw` 从缓存读到旧数据，塞给 LLM 当"当前事件线"，导致 LLM 只做微调复述。补上跟日程/大纲一致的 `localStorage.removeItem`。auto-advance 路径通过 `runGenerateLines(silent=true)` 直接调用，保留 `previousRaw` 做增量，不受影响。

### 2. Prompt 从"更新优先"翻转为"挖掘优先"
- 加"每次推演的核心任务顺序"框架：先挖新伏笔，再判归并，最后更新老的
- 归并规则改成两列平行判断（新建 vs 归并），判断不清默认新建
- 明确金句："宁可新建后再合并，也不要因为沾点边就塞进老事件线"
- 保留克制/禁止规则但柔化措辞（"必须"→"通常"，"严格禁止"→"避免"）

### 3. 叙事尺度分档
- 新增 per-character 叙事尺度设置（存 `extension_settings.scale[characterId]`）
- 4 选项：自动（默认）/ 宏观 / 中观 / 微观
- 微观档明确禁止势力/阴谋/战事/暴力冲突，冲突阶段重解释为心结/张力/摊牌/情感爆发
- 中观档聚焦家族/组织/职场，排除天下战争和纯两人情感
- 主 prompt 去掉默认宏观暗示，agency world 示例改为"对齐上方叙事尺度块"
- 设置 UI 加尺度 radio 组

### 4. 故事记忆库系统（core deliverable）
新文件 `memory.js`，350+ 行。架构（Plan C：单一客观记忆 + 视角化注入）：

- **L0 层**：每 N 楼（默认 5，可调 1-30）→ 1 段结构化摘要
- **L1 层**：每 M 段 L0（默认 10，可调 2-30）→ 1 章节摘要
- **存储**：`chat_metadata[MEMORY_KEY]`，服务端 chat 文件（不占浏览器缓存），随聊天迁移
- **延迟一楼**：最后一组永远处于"待定"，防重 roll
- **hash 变化检测**：group 内任一楼变了，整组 L0 作废重排队（覆盖 ST 事件不可靠场景）
- **失败阈值**：单组 3 次或连续 3 组失败 → 系统 paused，用户手动补齐/重构才恢复
- **NSFW 约束**：不记录具体细节，除非涉及承诺/真实伤害/身份揭示/怀孕/疾病

**结构化摘要字段**：
- L0: 时间锚点 / 场景 / 事件 / 人物（去掉了"伏笔"字段，避免注入时误导——真伏笔靠 LLM 写进事件描述保留）
- L1: 时间跨度 / 主要事件 / 关系变化

**时间处理**：优先绝对时间 YYYY-MM-DD，无则 fallback 相对天数"第N天"，两种不能混用。

### 5. 生成前记忆完整性检查
- 生成日程/大纲/事件线前扫 memory 健康度
- 有 pending 或 paused → 弹 confirm："记忆库不完整，仍要生成吗？"
- 用户可选"继续生成"（用当前不完整的记忆）或"取消"

### 6. 动态 loading 文案
- 生成时如果 memory 正在跑摘要 → "正在补全记忆并生成日程..."
- 否则 → "正在规划中..."
- 让用户知道当前处于哪个阶段

### 7. 视角化注入
`buildMessages()` 新增记忆库注入段：
- 全部 L1 章节 + 最近 6 段 L0
- 加"视角"标签让 LLM 按 user / char 视角解读同一份客观记忆
- 保留最近 10 楼原文（近景细节）
- 明确告知 LLM "优先信任记忆库描述"（因为记录了事件后的最新状态，可能与角色卡/世界书原描述冲突）

### 8. 设置面板"故事记忆库"分区
- 自动记忆开/关
- L0 分组楼数 / L1 章节段数 / 短楼跳过字数
- 健康度状态显示（AI 楼总数、稳定分组数、已生成 L0、待生成、永久失败、L1 章节数）
- 系统暂停提示 + 后台工作提示
- 进度条（重构 / 补齐时实时更新 current/total）
- 三个按钮：检查完整性 / 补齐缺失 / 推翻重构
